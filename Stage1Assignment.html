Basics of the Web & HTML

Part I: Documents of the Web

Types of Documents

Different types of files can be found on the web, including:

  • Images
  • Videos
  • Music
  • Plain Text
  • HTML (Hypertext Markup Language)

Of these different file types, HTML is the most prominent document on the web.

HTML documents, such as this one, are hosted on servers. In order to retrieve such documents, the internet uses HTTP to communicate with the user, the user's browser, and the server (see Figure 1).

Figure 1: Major Concepts of the Web

The Key Components of HTML

HTML is made up of text content (what one sees), markup (what it looks like), references to other documents (e.g., videos), and links to other pages (ex., see Wikipedia).

HTML Markups

HTML markups take the general format of <name>content</name>, called an element.

An element consists of an opening tag <name> and a closing tag </name>.

Below are some examples of tags:

  • <b>content</b>: used to add emphasis by bolding text
  • <em>content</em>: used to add emphasis by italicizing text
  • <a href="URL">content</a>: anchor used to link documents (a tag can have multiple attributes)
  • <img src="URL" alt="text">: use to add images like the one above
  • <p>content</p>: used to mark paragraphs (not a void tag)
  • <br>: used to add breaks / white space (a void tag)

Inline versus Block Elements

Block elements start (and end) with a new line, when displayed in a browser. Imagine a dotted lines around the text to delineate the line break.

Inline elements, on the other hand, are usually displayed without line breaks.

Block Element Examples
  • <br>
  • <span>
  • <a>
  • <img>
  • <strong>
Inline Element Examples
  • <form>
  • <p>
  • <div>

Part II: Creating a Structured Web Document

The Language Used to Create Structure

HTML provides the framework of a document and CSS and Javascript provide the needed style and interactivity to a web page.

In this course, the focus will be on HTML and CSS languages with the former used to control the "structure" of the web page and the latter the "style" of the page. The structure of HTML language, which consists of both syntax and rules, is tree-like and the DOM (Document Object Model) references this tree-like structure. HTML, as previously mentioned, consists of tags, which the browser uses to form the elements, and the nesting of elements within elements (a common feature of HTML; seen in the indentation of the text) gives way to the tree-like structure / DOM.

Structure can be given to the HTML language by the grouping of concepts using container elements <span> and <div>.

CSS is a stylesheet language used to describe the presentation or ordering of a document written in HTML. Importantly, CSS helps to mitigate repetition in code through its 'cascading' property.

An Overview of CSS: Cascading Style Sheets

The cascading property of CSS is an inheritence feature that relies on the ancestor-descendent relationship to operate. It is a means by which repetition in code is avoided. A helpful analogy is the application of national laws to states, municipalities and neighbourhoods (and, in turn, the application of state laws to municipalities and neighbourhoods, and so on). Through this mechanism the structure rules are applied not only to a specific element, by also to its descendents. As such, the property relies on a document tree or DOM of HTML.

Note: Generally, descendent elements inherit text related, but not box related properties.

CSS Selectors

Style can be applies to specific HTML elements through the use of the following selectors (for full list of selectors see href=https://css-tricks.com/how-css-selectors-work/>Treehouse™: Beginner Concepts: How CSS Selectors Work)):

  • ID Selectors: e.g., #abcdefg (most specific)
  • Class Selectors: e.g., .abcdefg (useful & versatile)
  • Tag Selectors: e.g., h2 (html specific)
  • Attribute Selectors and others (Positional Selectors etc.)

The Box Model

There are four key points to keep in mind about HTML and boxes:

  • HTML elements are boxes and each box has four components:margins, border, padding, and content
  • Because there are so many components to each box it can often be hard to get the size of the box just right
  • Two techniques help with the sizing issue: setting size in terms of percentages (as opposed to pixels), and setting box-sizing attribute to border-box for every element
  • Different browers work differently and can show CSS/HTML code differently

HTML & CSS Errors

Errors in HTML and CSS code are an inevitable part of content and web development. The process of web development is indeed an iterative one: code, test (debug), refine, and repeat. Common errors in HTML and CSS code include Syntax errors - e.g., missing semicolons, missing element end tags.

The errors can often be resolved by returning to the source code, but this may be a tedious process for larger documents. In such instances, the following W3C validation tools - W3C markup validation service and the W3C CSS validator - are beneficial for quickly identifying, at line level, the errors and warnings in HTML and CSS code. The recommended practice is to always validate HTML and / or CSS code.
